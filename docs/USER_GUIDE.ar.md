# StockBolt — دليل المستخدم

مرجع عملي يومي لتشغيل محل قطع غيار السيارات على StockBolt.
هذا الدليل يغطّي سير العمل الذي ستستخدمه فعليًا؛ ليس شاملاً.
للتقارير أو الإعدادات غير المذكورة هنا، التنقّل داخل البرنامج بديهي
بشكل عام — افتح القائمة المعنيّة واتبع تسلسل الأزرار.

> **نصيحة — الإجراءات السريعة**: زر **+** العائم في الزاوية السفلية اليسرى
> من كل صفحة يفتح قائمة بأكثر ست إجراءات إنشاءً شيوعًا (فاتورة، عرض سعر،
> دفعة، فاتورة مورد، إيصال استلام، منتج). استخدمه كاختصار من أي مكان.

---

## ١. تسجيل الدخول لأول مرة

إذا كنت تقرأ هذا، فقد أكملت معالج الإعداد ذا الست خطوات.
ملخّص سريع لما تمّ:

- إنشاء سجلّ **الشركة** (الاسم، الدولة، العملة، السنة المالية).
- زرع **دليل الحسابات** الخاص بدولتك (الإمارات/الخليج أو الهند).
- زرع **معدّلات الضرائب** القياسية و**طرق الدفع** و**وحدات القياس**.
- إنشاء **مستودعك** الأول (افتراضي).
- إنشاء حساب **بنكي / نقدي** أول.
- اختياريًا تحميل **بيانات تجريبية** (١٠ منتجات، عميلين، موردين).

إذا تخطّيت البيانات التجريبية، فالخطوة التالية هي إعداد
**البيانات الأساسية** قبل أن تتمكّن من تسجيل أي حركة.

## ٢. البيانات الأساسية

قبل تسجيل المبيعات أو المشتريات، تحتاج إلى:

### المنتجات (المخزون → جميع المنتجات → ‎+ منتج جديد)

لكل منتج، املأ:

| الحقل | لماذا يهمّ |
|---|---|
| **رمز المنتج (SKU)** | كود المخزون الداخلي لديك. يجب أن يكون فريدًا. |
| **الاسم** (إنجليزي + عربي) | ما يظهر على الفواتير وعروض الأسعار. |
| **رقم OE** | رقم الجزء الأصلي من الشركة المصنّعة — قابل للبحث في كتالوج القطع. |
| **العلامة التجارية**، **الفئة**، **توافق المركبات** | يستخدمها متصفّح كتالوج القطع. |
| **سعر البيع** | السعر الافتراضي على الفواتير. قابل للتعديل لكل سطر. |
| **حساب الشراء** | الحساب الذي يُرحّل إليه السطر في الدفاتر. الافتراضي **1300 المخزون** — اتركه كما هو للقطع التي تخزّنها. غيّره إلى حساب مصروف (5xxx/6xxx) لبنود مثل الرسوم الجمركية أو أجور النقل أو رسوم الخدمات. |
| **الحد الأدنى للمخزون** | إذا قلّ المخزون عن هذا الحد، يُفعَّل تنبيه نقص المخزون وتقرير إعادة الطلب. |

### العملاء (جهات الاتصال → العملاء → ‎+ إضافة عميل)

لكل عميل: الاسم، الهاتف، البريد الإلكتروني، الرقم الضريبي،
العملة، شروط الدفع (بالأيام؛ `0` = دفع عند التسليم)،
حد الائتمان، العنوان.

### الموردون (جهات الاتصال → الموردون → ‎+ إضافة مورد)

نفس حقول العميل؛ يُحدَّد النوع كـ `مورد` (أو `كلاهما` إذا كان الطرف
عميلاً وموردًا في آن واحد).

## ٣. تسجيل عملية بيع (التدفّق اليومي الرئيسي)

### أ. بيع نقدي سريع عبر نقطة البيع (POS)

للبيوع النقديّة أو بالبطاقة عند الكاونتر للعملاء العابرين:

1. **المبيعات → نقطة البيع** (أو من الشريط الجانبي: POS).
2. اضغط **افتح الجلسة** إذا لم تكن مفتوحة (أدخل الرصيد النقدي الافتتاحي).
3. ابحث عن المنتج (بالـ SKU أو رقم OE)، انقر لإضافته إلى السلّة.
4. عدّل الكميّة بأزرار +/-.
5. اختياريًا اختر **عميلًا** (تخطَّ للعميل العابر).
6. انقر **نقدًا** أو **بطاقة** أو **بالأجل** للتسوية.
7. في نهاية اليوم، **أغلق الجلسة** — أدخل النقد المحسوب فعليًا؛
   يسجّل النظام أي فروقات.

### ب. فاتورة لعميل آجل

1. **زر + → فاتورة جديدة** (أو المبيعات → الفواتير → ‎+ فاتورة جديدة).
2. اختر **العميل** (بحث ذكي — ابدأ بكتابة الاسم).
3. التاريخ يفترض اليوم؛ تاريخ الاستحقاق يتبع شروط دفع العميل.
4. أضف **بنود الأسطر**:
   - اختر المنتج من القائمة المنسدلة (بحث بالـ SKU أو الاسم).
   - عدّل الكميّة / سعر الوحدة / الخصم / الضريبة حسب الحاجة.
5. **احفظ** كمسوّدة.
6. **أكّد** عندما تكون جاهزًا. التأكيد يُرحّل:
   - قيد يومية مبيعات (مدين: ذمم العملاء، دائن: المبيعات، دائن: ضريبة القيمة المضافة المخرجات).
   - سطر في حركة المخزون (كميّة صادرة، تكلفة المنتج = متوسط التكلفة المتحرّك MAC).
   - قيد يومية تكلفة بضاعة مباعة (مدين: التكلفة، دائن: المخزون) — فقط إذا كانت التكلفة معروفة.
7. اطبع أو احفظ كـ PDF عبر زر **🖨 طباعة** في المحرّر.

### ج. استلام دفعة

1. **زر + → استلام دفعة** (أو المبيعات → المدفوعات → ‎+ دفعة جديدة).
2. اختر **العميل**.
3. **التصنيف:** "مقابل فاتورة" (افتراضي) يُظهر لوحة **تطبيق على الفواتير**
   — وزّع المبلغ على الفواتير المفتوحة (يملأ تلقائيًا الأقدم أولًا؛
   يمكنك تعديل أي صف).
4. اختر **الحساب البنكي** (حيث ستودَع الأموال).
5. أدخل **المبلغ**. تملأ اللوحة قيم التطبيق تلقائيًا.
6. احفظ ← أكّد. القيد المُرحّل:
   - مدين: البنك، دائن: ذمم العملاء، إضافة قيد دائن في "سُلف العملاء"
     إذا تجاوز المبلغ التخصيصات (الفائض يبقى كسلفة عميل يمكن تطبيقها لاحقًا).

## ٤. تسجيل عملية شراء

مساران حسب طريقة عملك:

### أ. مبسّط — البضاعة والفاتورة وصلتا معًا

1. **زر + → فاتورة مورد جديدة**.
2. اختر **المورد**.
3. أضف **بنود الأسطر** — اختر كل منتج، أدخل الكميّة وسعر الوحدة.
4. **احفظ** ثم **أكّد**.

ماذا يحدث عند التأكيد:

- لكل سطر منتجه ذو "حساب الشراء" = **1300 المخزون**:
  - يُضاف سطر في حركة المخزون (كميّة واردة، سعر الوحدة).
  - يُعاد حساب متوسط التكلفة المتحرّك (MAC) لذلك المنتج.
  - مدين: المخزون.
- للأسطر التي يكون حساب منتجها 5xxx/6xxx (رسوم جمركية، نقل):
  - مدين: ذلك الحساب مباشرة. لا حركة مخزون.
- مدين: ضريبة القيمة المضافة المدخلات للضريبة. دائن: ذمم الموردين للإجمالي.

### ب. الرسمي — أمر شراء ← إيصال استلام ← فاتورة مورد

للمحلات الأكبر التي ترغب بتتبّع أوامر الشراء:

1. **المشتريات → أوامر الشراء → ‎+ أمر جديد** — اختر المورد، أضف الأسطر،
   احفظ، **أرسل**.
2. عند وصول البضاعة: **المشتريات → إيصالات الاستلام → إيصال جديد** —
   يُملأ من أمر الشراء. أكّد لاستلام المخزون وتسجيل التزام مستحق.
3. عند وصول فاتورة المورد: **المشتريات → فواتير الموردين → فاتورة جديدة**
   — تُملأ من إيصال الاستلام. أكّد لتسوية الالتزام المستحق وتسجيل المستحقات.

### ج. دفع مبلغ لمورد

**المشتريات → مدفوعات الموردين → ‎+ دفعة جديدة**. نفس لوحة التطبيق
كمدفوعات العملاء، لكن تُطبَّق على فواتير المورد المفتوحة.

## ٥. إدارة المخزون

| الحاجة | المكان |
|---|---|
| رؤية الكميّة الحاليّة لمنتج | **المخزون → دفتر حركة المخزون** — فلتر حسب المنتج والمستودع |
| نقل مخزون بين المستودعات | **المخزون → التحويلات → تحويل جديد** |
| تعديل المخزون بعد جرد (مفقود، مكتشف، تالف) | **المخزون → التسويات → تسوية جديدة** |
| رؤية ما هو دون الحد الأدنى | **التقارير → تقرير إعادة الطلب** (أو انقر شريط تنبيهات نقص المخزون على لوحة التحكم) |
| إيجاد البنود بطيئة الحركة | **التقارير → المخزون بطيء الحركة** |
| تتبّع عمر المخزون | **التقارير → أعمار المخزون** |

### التسويات — متى تستخدم كل نوع

- **تسوية مخزون (ربح)** — وجدت مخزونًا لم تكن تعرف بوجوده.
  مدين: المخزون، دائن: 4300 ربح المخزون.
- **تسوية مخزون (خسارة)** — فقدت مخزونًا بسبب تلف / سرقة / انتهاء صلاحية.
  مدين: 6700 خسارة المخزون، دائن: المخزون.

## ٦. المرتجعات

### العميل يُعيد قطعة

1. **المبيعات → مرتجعات المبيعات → ‎+ مرتجع جديد** — اختر الفاتورة الأصليّة،
   استورد الأسطر، حدّد كميّة الإرجاع لكل سطر، احفظ.
2. ينشئ النظام **إشعار دائن** تلقائيًا (المبيعات → إشعارات دائنة).
3. أكّد إشعار الدائن. هذا يعكس قيد البيع الأصلي ويُعيد البضاعة للمخزون
   (إذا كانت قابلة لإعادة البيع).
4. الرصيد الدائن يبقى كسلفة عميل — طبّقه على فاتورة مستقبليّة عبر
   تدفّق الدفع.

### إرجاع قطع إلى مورد

1. **المشتريات → إشعارات مدينة → ‎+ إشعار جديد** — اختر المورد والفاتورة
   الأصليّة، حدّد كميّة الإرجاع لكل سطر، احفظ.
2. أكّد. يعكس عمليّة الشراء الأصليّة: المخزون يخرج، ذمم الموردين تنخفض.

## ٧. المعاملات البنكيّة

### التحويلات البنكيّة
**المحاسبة → التحويلات البنكيّة → ‎+ تحويل جديد.** اختر حساب "من"
وحساب "إلى"، أدخل المبلغ. حركة محايدة محاسبيًا.

### مصاريف يوميّة (إيجار، خدمات، إلخ)
**المحاسبة → المصاريف → ‎+ مصروف جديد.** اختر حساب المصروف (6xxx)
والحساب البنكي الذي سيُدفَع منه، والمبلغ.

### الشيكات الآجلة (PDC)

إذا دفع عميلك بشيك بتاريخ مستقبلي، أو أعطيت أنت شيكًا لمورد:

- **المحاسبة → شيكات مستلمة** — سجّل شيكًا من عميل. عند إيداع الشيك،
  حدّده "مُودَع"؛ عند صرفه، حدّده "محصّل". الشيكات المرتدّة تُعاد إلى
  حساب "شيكات مرتجعة" المستحقّة القبض.
- **المحاسبة → شيكات صادرة** — نفس الفكرة للشيكات التي تعطيها للموردين.

## ٨. التقارير التي ستنظر إليها فعليًا

| التقرير | ما الذي يُجيب عليه |
|---|---|
| **لوحة التحكم** | مبيعات اليوم، مشتريات اليوم، قيمة المخزون، الذمم المدينة والدائنة، النشاط الأخير. الشاشة الرئيسيّة. |
| **الأرباح والخسائر** | هل تربح هذا الشهر؟ يعرض المبيعات − تكلفة البضاعة المباعة = إجمالي الربح، ثم ناقص المصاريف التشغيليّة = صافي الربح. |
| **الميزانية العموميّة** | ماذا تملك مقابل ما عليك في الوقت الحالي؟ الأصول المتداولة، الأصول الثابتة، الالتزامات المتداولة وطويلة الأجل، حقوق الملكيّة، و**رأس المال العامل** (الأصول المتداولة − الالتزامات المتداولة). |
| **أعمار الذمم المدينة (AR Aging)** | من يدين لك بالمال ومنذ متى. بأقسام: حالي / 31–60 / 61–90 / أكثر من 90. |
| **أعمار الذمم الدائنة (AP Aging)** | لمن تدين بالمال ومنذ متى. |
| **تقييم المخزون** | كم تساوي بضاعتك الآن (بمتوسط التكلفة المتحرّك). |
| **تقرير إعادة الطلب** | ماذا تشتري بعد ذلك — أي شيء عند أو تحت حدّه الأدنى للمخزون. |
| **إقرار ضريبة القيمة المضافة** | ملخّص ربعي لضريبة القيمة المضافة في الإمارات/الخليج (أو GST للهند). |
| **سجلّ المراجعة (Audit Log)** | من فعل ماذا ومتى. كل تأكيد / إلغاء / تعديل. |

### عرض ٣٦٠ للعميل والمورد

انقر أي اسم عميل أو مورد في القائمة لفتح **عرض ٣٦٠** الخاص به.
مؤشّرات أداء (المعلّق، المتأخّر، الإجمالي خلال ١٢ شهرًا، عدد المستندات)،
رسم بياني للأعمار، وعلامات تبويب لكل مستنداته المفتوحة والتاريخ
وكشف الحساب الكامل. كشف الحساب قابل للطباعة وتصدير PDF.

## ٩. إقفال الفترة

مرّة شهريًا / ربع سنوي، أقفل الدفاتر لمنع تعديل الفترات السابقة:

1. **المحاسبة → قفل الفترة**.
2. اضبط تاريخ القفل على اليوم الأخير من الفترة (مثلاً `2026-04-30`).
3. بعد ذلك، لن يمكن ترحيل أي قيد يومية أو فاتورة أو دفعة بتاريخ ≤
   تاريخ القفل. لترحيل قيد تصحيحي لفترة مقفلة، انقل تاريخ القفل
   إلى الأمام مؤقّتًا.

## ١٠. نصائح

- **البحث** — كل قائمة منسدلة للعميل / المورد / المنتج تدعم البحث الذكي.
  ما عليك سوى أن تبدأ بالكتابة.
- **الإشعارات** — الجرس في الزاوية العلوية اليسرى يُظهر: الفواتير
  المتأخّرة، البنود ذات المخزون المنخفض، المدفوعات المستلمة اليوم.
  انقر أي إشعار للقفز مباشرة إلى المستند.
- **الطباعة** — كل مستند قابل للطباعة (فاتورة، عرض سعر، أمر شراء،
  إيصال استلام، فاتورة مورد، إشعار دائن، إشعار مدين، كشف حساب)
  له زر **🖨 طباعة**. تُفتح المخرجات في تبويب جديد بتنسيق جاهز للطباعة؛
  أمر متصفّحك "طباعة ← حفظ كـ PDF" يُكمل الباقي.
- **إعدادات الطباعة** — **الإعدادات → إعدادات الطباعة** تتيح لك تخصيص
  ترويسة الشركة ولون التمييز ونص التذييل لكل المطبوعات.
- **اللغات** — بدّل بين الإنجليزيّة والعربيّة في أي وقت عبر زرّ اللغة
  في الشريط العلوي. تنقلب الواجهة بالكامل إلى RTL عند اختيار العربيّة.
- **سلامة البيانات** — **الإعدادات → فحص النظام** يُشغّل القيود التسع
  (ميزان المراجعة متوازن، أعمار الذمم المدينة = حساب 1200، أعمار الذمم
  الدائنة = 2100، إلخ). إذا تحوّل أي شيء إلى أحمر، اتصل بالدعم قبل
  متابعة الترحيل.
