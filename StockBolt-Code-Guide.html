StockBolt — Your Code Folder Guide

A plain-English map of where everything lives, so you always know where to go. Made for re-checking anytime.

The one sentence to remember: to change a feature, open src/modules/<that area>/. Everything else is supporting cast.

1. The big idea — your app is 3 layers

Everything you’d ever edit lives inside one folder: src (short for “source code”). Inside it, picture 3 stacked layers:

2. The folder tree (a picture of it)

src/ ├─ modules/ ← your screens (the big one) ├─ components/ ← shared pieces (top bar, pickers…) ├─ ui/ ← buttons, tables, colours ├─ i18n/ ← all on-screen wording (English + Arabic) ├─ data/ ← talks to the database ├─ store/ ← remembers who’s logged in ├─ hooks/ lib/ core/ ← reusable logic helpers ├─ types/ ← shapes of your data ├─ App.tsx ← the master list of all pages + their web addresses └─ main.tsx ← boots the app (almost never touched)

6. Worked examples (with before → after)

Example A — Add a row to the Settings gear menu

File: src/components/app-layout.tsx → find const settings.

BEFORE 
const settings: NavItem[] = [
  { to: '/settings/company', label: 'Company' },
];
AFTER — one new line added 
const settings: NavItem[] = [
  { to: '/settings/company', label: 'Company' },
  { to: '/settings/users',   label: 'Users & Roles' },
];

Example B — Change a word anywhere in the app

File: src/i18n/en.json. Change only the text on the right of the colon.

BEFORE 
"save": "Save"
AFTER 
"save": "Save changes"

BEFORE — a button that opens a menu 
<button onClick={() => setOpen(!open)}>
  <CogIcon />

{open && ( ...the dropdown menu... )}
AFTER — a plain link straight to Settings 
<Link to="/settings">
  <CogIcon />
</Link>

7. How to edit a file safely (the 4 keys)

StepDo this
Open a filePress Ctrl + P, type the file name (e.g. app-layout), press Enter
Find the spotPress Ctrl + F, type what you’re looking for (e.g. const settings)
SavePress Ctrl + S — the app reloads by itself
Undo a mistakePress Ctrl + Z a few times to go back

🔢 Your live safety gauge: the bottom-left of the editor shows an error count. As long as it says 0 after you save, your code is structurally fine. If a number appears, press Ctrl+Z until it’s 0 again.

8. The golden safety rule

🟢 Safe to poke: modules/, components/, ui/, i18n/. These only draw what you see. The worst that can happen is a typo your editor flags instantly — and Undo fixes it. They can never corrupt invoices, stock, or accounting.

🔴 Be careful — ask first: anything under supabase/migrations/ (your database) and the accounting / posting logic. Those touch real money & stock. When in doubt, don’t change them alone.
